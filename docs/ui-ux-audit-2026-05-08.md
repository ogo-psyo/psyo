# PSO MVP — UI/UX audit

## Вердикт

Текущее приложение уже доказывает продуктовую идею, но UX ощущается как набор фич в тесном корпусе телефона. Главная проблема не визуальный стиль сам по себе, а отсутствие ясной иерархии: onboarding, Today, паспорт, ассистент, карта и список спорят за внимание.

## Что ломает ощущение качества

1. **Нет одного главного сценария**
   - Пользователь не понимает, что важнее: создать героя, заполнить паспорт, спросить ассистента или добавить care-loop.
   - Экран Today пытается быть dashboard, onboarding follow-up и task manager одновременно.

2. **Паспорт перегружен**
   - Слишком много полей одним потоком.
   - Нет progressive disclosure: редкие/медицинские/поведенческие поля должны раскрываться блоками.
   - Порода раньше была dropdown-ловушкой; поиск уже добавлен, но экран всё ещё длинный.

3. **Аватар обещает больше, чем бесплатный fallback может дать**
   - Формулировки создают ожидание similarity по фото.
   - На бесплатном Pollinations лучше позиционировать как “стилизованный герой по породе и признакам”, а не точная генерация по фото.

4. **Визуальная система слишком шумная**
   - Acid/cyan/violet/glass/emoji/карточки/капсулы везде одновременно.
   - Много одинаково ярких элементов, мало спокойных зон.
   - На маленьком экране это быстро превращается в “игровую панель”, но без игровой структуры.

5. **Навигация конфликтует с контентом**
   - Bottom tabs есть, но экран всё равно выглядит как длинная страница.
   - Header + hero card + tabs съедают слишком много вертикали.

6. **Копирайт местами умнее интерфейса**
   - Хорошие идеи: “care-loop”, “цифровой мир собаки”, “не общий чат”.
   - Но их слишком много, и они не сложены в простой user journey.

## Рекомендованная новая IA

```text
App shell
┌─────────────────────────────┐
│ Hero dog card / status       │
├─────────────────────────────┤
│ Today: 1 main action         │
│ + 2 secondary cards          │
├─────────────────────────────┤
│ Tabs: Today / Dog / Ask / Map│
└─────────────────────────────┘
```

### Tabs

1. **Today**
   - one next-best-action
   - care-loop tasks
   - small assistant prompt

2. **Dog** вместо “Паспорт”
   - avatar
   - breed/profile essentials
   - expandable sections: Health, Behavior, Social, Admin

3. **Ask** вместо “Ассистент”
   - чат + 3 сценария
   - context chips: breed, age, energy, health state

4. **Map**
   - zones and places
   - social later

5. **Things** можно оставить позже, не в первичном MVP nav.

## Приоритетные fixes

### P0 — немедленно
- Переименовать “Паспорт” → “Собака” или “Профиль”.
- Сократить hero/header высоту на внутренних экранах.
- В аватаре заменить обещание “по фото” на “по породе, признакам и фото как reference”.
- Разбить профиль на essentials + details.

### P1 — после справочника пород
- Сделать Breed Directory полноценным компонентом: search → breed card → select.
- Добавить “визуальный рецепт” перед генерацией: порода, шерсть/кожа, размер, цвет, стиль.
- Assistant context chips перед вопросом.

### P2 — polish
- Уменьшить количество neon accents.
- Ввести 3 уровня карточек: primary / neutral / quiet.
- Стабилизировать spacing scale: 8/12/16/24.
- Убрать лишние emoji там, где они конкурируют с аватаром.

## Следующий дизайн-slice

**Dog Profile 2.0**:
- верх: avatar + имя + breed search
- essentials: возраст, размер, шерсть, энергия
- breed card из справочника
- collapsed sections: Health / Behavior / Social
- CTA: “Собрать героя” + “Спросить по профилю”

Это даст максимальный прирост качества без переписывания всего MVP.
